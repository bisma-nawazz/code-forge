# Code Review
## Instructions
1. Read the full diff or file — do not skim. Understand what the code is trying to do before critiquing it.
2. Check for logic bugs: off-by-one errors, incorrect conditionals, unhandled edge cases, and race conditions.
3. Flag unclear variable names, overly complex functions, and missing abstractions that hurt readability.
4. Identify style violations relative to the project's established conventions.
5. For every issue, explain why it matters and what the correct fix should be — not just what's wrong.
