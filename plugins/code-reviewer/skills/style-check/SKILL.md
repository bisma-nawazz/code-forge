# Style Check
## Instructions
1. Identify deviations from the project's naming conventions, file structure, and formatting standards.
2. Flag N+1 query patterns, unnecessary re-renders, blocking I/O in async contexts, and other performance anti-patterns.
3. Check for unnecessary complexity: deeply nested conditionals, overly long functions, and redundant code.
4. Suggest specific rewrites, not vague advice — show before and after for each issue.
5. Distinguish between hard violations (must fix) and stylistic preferences (nice to have) in your output.
