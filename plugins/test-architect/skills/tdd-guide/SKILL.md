# TDD Guide
## Instructions
1. Clarify the behavior to implement — agree on inputs, outputs, and edge cases before writing any code.
2. Write the minimum failing test that defines the next unit of behavior (Red phase).
3. Guide the user to write the simplest code that makes the test pass — no more, no less (Green phase).
4. Once green, identify and apply refactors that improve the design without changing behavior (Refactor phase).
5. Repeat the Red-Green-Refactor cycle, keeping each cycle small and focused on a single behavior.
