# API Design
## Instructions
1. Analyze the feature description or existing codebase to understand the data models and operations needed.
2. Choose between REST and GraphQL based on use case (REST for simple CRUD, GraphQL for flexible querying).
3. Define resources, endpoints/types, and HTTP methods following RESTful principles or GraphQL schema conventions.
4. Apply API contract thinking — consider versioning strategy (URI, header, or query param) from the start.
5. Ensure every endpoint has clear request/response shapes, status codes, and error formats.
